1. 「開発中」と「完成後」のサーバーの役割の違い
開発中（npm run dev）：
開発専用の、超高機能なlocalhost（ローカル）Webサーバー。
ブラウザが読めないTypeScriptを、その場で超高速で翻訳して届ける「翻訳マシン付きのサーバー」として動いていた。

完成後（npm run build）：
すでに翻訳済みの「ただのHTMLとJSの束（静的ファイル）」に変換される。
本番サーバーはプログラムを実行するのではなく、ただその完成品を配るだけの郵便局になる
これを「静的サイトホスティング」という。


2. ブラウザのセキュリティ（CORS制限）の壁
現代のWebアプリ（Reactなど）は、効率化のために裏で複数のファイルを読み込み合う仕組みになっている（type="module"）。

しかしブラウザは、パソコン内のファイル（file://）が勝手に他のファイルを読み込むのを「ウイルスかもしれないから危険！」と
本気で拒絶する。
セキュリティの観点からブラウザは、「インターネットからダウンロードしたわけでもない、パソコン内のローカルファイル（file://）が、
勝手に別のローカルファイルを裏で読み込んで実行してはいけない」という非常に厳しいセキュリティルール「CORS(コルス)制限」がある。
もしCORS制限がないと、悪意のあるHTMLファイルをあなたがダブルクリックして開いただけで、そのHTMLのJavaScriptがあなたのパソコン内の
重要なファイル（写真やパスワードなど）を裏で勝手に読み込んで盗み出せてしまう危険がある

だから、ファイル自体は完璧にできていても、index.htmlダブルクリックのローカル実行(→file://での起動)だと画面が真っ白になる。
⇒ブラウザのセキュリティセンサに引っ掛かっているだけで、ちゃんと作れば物自体は完成できる。
動きを見たいときには諦めてlocalhostのnpm run devで確認する


3. なぜ「1つのJS」にまとめないのか？（esbuild と Vite の思想差）
旧来のやり方（esbuild --bundle）：
すべてのtsファイルを完全に溶かして1つのjsファイルにまとめる。裏でのファイル読み込みが発生しないため、index.htmlダブルクリックの
ローカル実行(→file://での起動)でも動いていた。

現代のやり方（Viteの標準設定）：
アプリのソースコードが1000万行など複雑かつ巨大になったとき、1つのjsにまとめると「開発時のビルド待ち時間が重い」
「スマホの読み込みが重い」という限界が来る。そのため、あえてファイルを機能ごとに小分けにして配る（コード分割）
⇒ローカル実行を諦める。
結論： 現代は、「ローカル実行の手軽さ（単一化）」を犠牲にして、「巨大アプリでも爆速（分割化）で動かす」というメリットをとった。



●まとめ
開発手順
# 1. Webアプリのフォルダに入る
cd my-game

# 2. my-game/vite.config.ts を開き、base を以下のように書き換える
export default defineConfig({
  plugins: [react()],
  base: './', // 「./（同じフォルダ内）」という相対パスの指定に変え、そのプロジェクトのディレクトリからファイルを参照しに
行くようになる
})

# 3. 開発サーバーを起動する（ローカル確認用）
npm run dev

# 4. 完成したからWebサーバー(GitHub Pages)で世界公開するためのリリース作業
npm run build

※エラーや動かないときはnpm installでnpmをリセットする

# 5. 本番サーバーにアップロード


今回のこの一連の流れは、現代のWebエンジニアが毎日仕事でやっていることと全く同じです。
①npm run dev で、自分のパソコン（localhost）だけでこっそり動かして開発する。
②完成したら npm run build で、本番用の最高に綺麗なファイル一式（dist）を書き出す。
③ローカルで直接開くとセキュリティで弾かれるのは知っているので、そのまま 本番Webサーバーにアップロード する。
④本番のURL（https://...）で、世界中の人と一緒に正しく動くか確認する。
⑤プロの開発者も、ビルドした後の dist/index.html をダブルクリックして確認したりはしません（動かないと分かっているからです）。
  みんな、Webサーバーに載せてから動作確認をします。


ちなみに、本番相当の動きを確認する以下のコマンドがある。
これを叩くと、Viteが「dist フォルダ専用の臨時のlocalhostサーバー」が立ち上がる。
= CORS制限をすり抜けた、本番と全く同じ状態
npm run preview
